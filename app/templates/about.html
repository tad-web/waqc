About WAQC

The Web Accessibility Quick Checker, or WAQC, is a webapp that will check a website for some basic accessibility violations. Unlike many other accessibility checkers, WAQC itself is built to be as accessible as possible. The interface is simple, the output is not meant solely for individuals with technical backgrounds, and all parts of this tool are accessible to users with screen readers. We built this for and with our community partner, the Carroll Center for the Blind, in the context of a project for the Technology, Accessibility, and Design course at Olin College of Engineering.

WAQC's purpose is two-fold. Primarily, it serves as a digital tool to improve the Carroll Center's ability to more quickly and automatically inspect the accessibility of a defined set of elements on any website. This tool will enable our Community Partner to reach a larger volume of potential clients more rapidly and cost-effectively. WAQC's second, but perhaps equally important purpose, is to act as a medium through which we, as apiring engineers, can learn more about the accessibility of technology in the world around us and perhaps serve as a positive example for teaching others about the importance of accessibility.

If you want to help WAQC grow, check out our source code on Github!

Process

We worked closely with our community partner, the Carroll Center for the Blind, to generate ideas, select some appropriate accessibility barriers for our tool to check, and make sure our tool is accessible as we built it. We met weekly and then nearly every other day to run through the current state of the tool and make decisions together for next steps. Because we so closely co-designed this tool with the Carroll Center, many of our decisions are tailored to their specific use case, but we hope WAQC can be useful to others as well. This entire process, from design to implementation, was completed from March to May of 2018.

We created a web application in python using primarily Flask and Jinja2, and hosted it with Heroku. Please check out our source code on Github for more detailed implementation information and instructions to install and run our project locally.

Partners

Throughout this project, Bruce Howell, the Carroll Center's Accessibility Services Manager, served as our liaison to the Carroll Center. The Carroll Center evaluates websites and mobile applications for their clients to help improve accessibility for persons who are blind or otherwise disabled. Since The Carroll Center, as a blindness rehabilitation agency, teaches individuals who are blind or visually impaired to use assistive technologies, they are also commited to helping those who design and build websites and mobile applications to make them more accessible. We were lucky to have so much of the Carroll Center's time and collaboration in this project and are thankful for the ability to co-design the website with them.

This accessibility checker tool is a project for our course Technology, Accessibility, Design at Olin College of Engineering in the Spring 2018 semester. We were supported by our two professors, Caitrin Lynch and Paul Ruvolo. Together, they collaborated on this course, Technology, Accessibility, Design, first offered in Spring 2018.
